# Engineering Portfolio v0.1

## Purpose

The Engineering Portfolio is an evidence-backed record of engineering competence acquired through Workbench.

It does not record technologies studied.

It records engineering capabilities that have been practiced.

Every entry must reference verifiable evidence contained within the repository.

---

# Principles

* Practice before confidence.
* Evidence before claims.
* Breadth through deliberate exploration.
* Depth through repeated practice.
* Engineering principles over specific technologies.
* Confidence must be earned through implementation.

---

# Confidence Scale

| Level | Meaning                                                       |
| ----- | ------------------------------------------------------------- |
| 0     | No practical experience.                                      |
| 1     | Practiced by following guidance.                              |
| 2     | Understands the implementation and can explain it.            |
| 3     | Can reproduce independently.                                  |
| 4     | Can teach and review others' implementations.                 |
| 5     | Can design, justify, and evolve the capability independently. |

Confidence should increase only after additional evidence is produced.

---

# Evidence Requirements

Evidence may include:

* Git commits.
* Pull requests.
* Completed experiments.
* Implemented features.
* Test suites.
* Infrastructure deployments.
* Production deployments.
* Architectural decisions.
* Operational incidents.
* Refactoring exercises.

Every portfolio entry must reference at least one piece of evidence.

# Portfolio Completion Criteria

The portfolio is considered complete when:

* Every targeted engineering capability has supporting evidence.
* Confidence ratings accurately reflect demonstrated competence.
* The evidence spans the complete software lifecycle from design through production operations.
* The repository provides sufficient evidence for another engineer to independently assess the claimed competencies.
